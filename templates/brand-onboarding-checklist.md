# Brand Onboarding Checklist

Use this when bringing a new brand, product line, or acquired design library into the standard.

- create `DESIGN.md` and `BRAND.md`
- define canonical naming rules and asset taxonomy
- add core tokens for color, type, space, radius, and motion
- add theme files for light, dark, and any accessibility modes
- add manifests for logos, social banners, and primary campaign assets
- add asset docs for every approved non-trivial asset
- assign owners and approvers in manifests and `CODEOWNERS`
- run validation and generate a catalog
- publish an initial tagged release
